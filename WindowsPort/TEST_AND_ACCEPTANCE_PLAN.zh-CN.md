# 测试与验收计划

Language: [English](TEST_AND_ACCEPTANCE_PLAN.md) | 简体中文

最后更新：2026-05-26

## 构建验证

- 干净构建成功。
- 单元测试通过。
- 应用可通过开发命令启动。
- 打包后应用可启动。

## 必备样例文件夹

请创建或收集：

- 仅 JPEG 文件夹。
- 仅 RAW 文件夹。
- RAW+JPEG 同名文件夹。
- 垂直 DNG 图片。
- `12000x6000` 或类似尺寸的全景图片。
- 非标准比例图片。
- 包含已生成代理的文件夹。
- 代理缺失或过期的文件夹。
- Lightroom XMP 样例。
- Capture One XMP 样例。
- 大文件夹：200+ RAW。
- 更大文件夹：如条件允许，500-1000 RAW。

## 几何验收

对每个样例检查：

- 主预览图像边界与实际可见图像一致。
- 裁切遮罩完全覆盖裁切区域外部。
- 左、右、上、下没有 1px 亮缝。
- 辅助线不超出图像，除非它们明确绑定到裁切矩形。
- 胶片条边框贴合实际缩略图内容。
- 全景图不会得到过大的边框。
- 垂直 RAW 不会继承横向元数据边框。
- 100%、125%、150%、200% DPI 缩放不会重新引入缝隙。

## 性能验收

应测量：

- 初始扫描时间。
- 首批缩略图出现时间。
- 首张画廊预览出现时间。
- 方向键导航延迟。
- 扫描 200/500/1000 RAW 后的内存占用。
- 代理生成吞吐。
- 复制吞吐。
- 哈希校验成本。

目标值应在首个 Windows 原型之后继续细化。

## 元数据/XMP 验收

- 读取评分。
- 读取 Pick/Reject。
- 读取颜色标签。
- 写入评分。
- 写入 Pick/Reject。
- 写入颜色标签。
- 保留未知 XML。
- 不破坏格式异常的 sidecar。
- Lightroom 往返。
- Capture One 往返。

## 复制验收

- 复制选中 JPEG。
- 复制选中 RAW。
- 复制 RAW+JPEG 配对。
- 仅复制 RAW。
- 日期文件夹规则。
- 保留目录结构规则。
- 相机文件夹规则。
- 评分文件夹规则。
- 冲突自动重命名。
- 冲突跳过。
- 冲突覆盖。
- 文件间暂停。
- 暂停后继续。
- 取消时不破坏已完成文件。
- 数量/大小校验。
- SHA-256 校验。

## UI 验收

- 左侧边栏在最小宽度下仍可用。
- 源目录和目标目录分支行为清晰。
- 筛选可发现、可重置。
- 工具栏复制与复制设置位置接近。
- 画廊胶片条可折叠。
- 检查器在元数据很长时可滚动。
- 深色和浅色模式均可读。
- 快捷键在合适场景下不受焦点影响。

## 发布验收

- 安装器可安装应用。
- 应用在没有开发环境的机器上可启动。
- 设置可持久化。
- 缓存文件夹位于正确的 Windows 用户数据位置。
- 应用可干净卸载。
