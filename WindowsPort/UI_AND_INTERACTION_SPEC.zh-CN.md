# 界面与交互规格

Language: [English](UI_AND_INTERACTION_SPEC.md) | 简体中文

最后更新：2026-05-26

## 主窗口布局

Windows UI 应保留 macOS 应用的工作区结构：

```text
工具栏
  复制 | 复制设置 | 更多 | 搜索

左侧边栏
  相机卡状态
  源文件夹
  源子文件夹
  目标文件夹
  目标子文件夹
  筛选

主浏览区
  头部：标题、可见数量、视图模式、排序方向、排序字段
  内容：网格 / 列表 / 画廊

右侧检查器
  预览元数据
  评分
  Pick 状态
  颜色标签
  Scopes
  EXIF 详情

底部画廊区域
  工具条：直方图、波形、辅助线、裁切、旋转/翻转
  胶片条
```

## 视觉风格

- 以工作为中心，信息密度较高。
- 深色模式必须优秀。
- 滚动条、菜单、按钮和检查器面板应使用一致的深色中性控件语言。
- 背景使用中性色：黑、白、深灰、中灰、浅灰、系统色。
- 避免营销式卡片和装饰性 hero 布局。
- 使用清晰的图标按钮。
- 重复文件项可以采用卡片式，但不要使用层层嵌套的装饰卡片。

## 视图模式

### 网格/图标视图

- 自适应缩略图网格。
- RAW+JPEG 折叠堆叠显示。
- 评分、Pick、颜色标签可见。
- Space 打开预览。
- 方向键移动选择。
- Ctrl+A 选择当前可见文件。

### 列表视图

- 行内显示小缩略图和元数据列。
- 显示排序指示。
- 选择行为和快捷键与网格视图一致。

### 画廊视图

- 主图预览。
- 右侧检查器应是主窗口全高右列，而不是只属于画廊视图的子面板。
- 底部胶片条。
- 胶片条条目边框必须匹配实际缩略图内容比例。
- 裁切遮罩和辅助线必须匹配实际渲染图像矩形。

## 键盘快捷键

Windows 可使用 Ctrl 替代 Cmd：

- Ctrl+A：选择当前可见项目。
- Esc：清除选择或关闭临时 UI。
- Space：预览。
- Ctrl+1/2/3：切换视图模式。
- 0-5：评分。
- P：Pick。
- X：Reject。
- U：Unmarked。
- 方向键：导航。
- Delete：根据上下文标记拒绝或移除，但默认避免破坏性删除。

## 裁切与辅助线行为

- 辅助线：三分线、中心线、对角线、黄金比例。
- 裁切比例：隐藏、原图、1:1、4:3、3:2、16:9、5:4、9:16、自定义。
- 裁切样式：遮罩或边框。
- 遮罩必须由独立矩形构成，并带少量外扩 bleed。
- 线条应细、低干扰。

## Scopes

- 直方图和波形应支持浮动显示和检查器内显示。
- 浮动面板应把设置保留在面板 chrome 内，使设置与图表一起移动。
- 直方图和波形应共享一致的面板 chrome、标题栏、位置控制和紧凑设置 chip。
- 必须显式测试浅色模式可读性。

## Source 与 Target 分支

- Source 和 Target 卡片展示已选择的根目录以及当前活动分支。
- 点击子文件夹只改变活动分支，不应替换根目录。
- 源扫描使用当前源分支。
- 复制目标使用当前目标分支。
- 保留目录结构的复制规则仍从源根目录计算相对路径。

## 本地化与设置

- 语言选项：系统、英文、简体中文、德文。
- 顶部工具栏显示选项：仅图标、仅文字、图标加文字。
- 低频设置应放在专门的 settings 区域或 More 菜单中，不要散落在无关工作流区域。

## 复制设置

复制按钮应在视觉上靠近复制设置。macOS 版本将复制设置移出侧边栏，是因为距离过远会让流程显得割裂。Windows 版本应保留这一原则。

## 审片环境

设置应包含：

- 应用外观：系统、浅色、深色。
- 预览背景：系统、黑、白、深灰、中灰、浅灰。
- 审片留白：无、小、中、大。

软打样是后续功能，应在明确的色彩管理渲染路径上实现。
