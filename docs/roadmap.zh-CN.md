# 路线图与已知缺口

语言：[English](roadmap.md) | 简体中文

本文用于跟踪 Windows 移植版相对 macOS 参考实现的状态。

## 近期优先级

### P0：核心工作流可靠性

- 保持 XMP sidecar 读写与常见摄影工具兼容。
- 在复制、重命名、覆盖和自动重命名流程中保留 sidecar。
- 改进 RAW、视频和损坏文件的预览失败反馈。
- 保持大文件夹扫描响应性。
- 修改复制、XMP、元数据或配对逻辑前优先补测试。

### P1：跨平台 UI 一致性

- 持续对齐 Windows 与 macOS 的工具栏、侧边栏、画廊、检查器和状态反馈。
- 改进快捷键一致性。
- 持久化更多布局状态，包括浮动面板位置。
- 为源目录和预览增加更清晰的空状态、加载状态和错误状态。

### P1：相机卡工作流

- 保留 DCIM 检测和自动选择行为。
- 通过 Windows 设备 API 实现安全弹出，或继续保留当前手动安全移除降级。
- 使用可注入 drive abstraction 为可移动盘检测增加测试。

### P2：预览和媒体兼容性

- 用更可靠的图像后端替换或补充 `System.Drawing` 预览路径。
- 通过 WIC、Microsoft Raw Image Extension、LibRaw 或代理图提升 RAW 兼容性。
- 增加视频预览/代理处理。
- 当预览来自代理或不可用时，在 UI 中给出更清晰提示。

### P2：打包

- 增加可重复运行的发布脚本。
- 增加 GitHub Actions Windows 构建/测试。
- 工作流稳定后考虑安装包。

## 当前已知限制

- Windows 安全弹出尚未实现。UI 会给出安全降级提示，而不是执行 shell 命令。
- RAW 预览支持依赖 codec 和代理图；不能保证所有 RAW 格式都能渲染。
- 目录树浏览刻意保持浅层，避免 UI 卡顿。
- 浮动直方图/波形图的位置尚未完整持久化。
- Windows UI 已接近 macOS 工作流，但并非像素级一致。
- `System.Drawing.Common` 当前仍会产生包警告，后续应重新评估。

## 不要做的事

- 不要用 macOS 源码或其它 Windows 工作区整体覆盖当前 Windows 代码库。
- 不要提交构建产物、发布 zip 或嵌套对照工作区。
- 不要通过插值用户路径的 PowerShell 命令实现设备弹出。
- 修改复制或重命名行为时，不要静默丢弃 sidecar 处理。
